# Todo · #apply 段落重设计（收敛到参赛者旅程）

> 版本：v1 · 创建于 2026-07-06
> 目标：把首页 #apply 段落从"幕后管理 + 参赛者操作台"混合视角，收敛为纯参赛者旅程。
> 用户决策：邮件审核/AI 标签/飞书备份 等幕后动作完全从首页移除（已有独立后台系统）。

## 问题分析

现状视角混乱：
- 标题"BEHIND THE STAGE / 台前是火种，幕后是流程" — 管理/运营视角
- system-flow 6 格混合：报名/作品/评分（参赛者）+ AI 标签/审核通知/飞书备份（管理者）
- demo-console "Interactive Demo / 参赛流程控制台" — 参赛者视角，但和上面"幕后"概念矛盾
- 调试文案外露："未登录预览" / "登录后可提交报名；未连接后端时会进入本地 Demo 模式" / "提交报名 Demo" / "登录 / 切换账号"
- "可编辑 / draft / public / sync / drive / retry" 等技术 badge

## 新设计

### 标题与叙事
- eyebrow：`YOUR 48 HOURS`（替代 `BEHIND THE STAGE`）
- h2：`从报名到展示，你只需要走完这四步。`（替代"台前是火种，幕后是流程"）
- 描述：站在参赛者口吻，告诉他在 48 小时里会经历什么

### system-flow 4 格（替代 6 格）
1. 报名
2. 现场冲刺
3. 提交作品
4. 展示评分

去掉的：AI 标签 / 审核通知 / 飞书备份（迁到独立后台）

### demo-console
- 头部："参赛流程预览" 替代 "Interactive Demo / 参赛流程控制台"
- 状态文案：未登录时显示"游客预览"，去掉"未登录预览"冰冷感
- tabs：3 个（报名 / 作品 / 评分），去掉"飞书"
- 内容去掉所有 "draft / public / sync / drive / retry" 技术 badge，改成口语化描述
- AI 整理行：保留但改成"提交后会自动整理一份报名摘要，方便现场签到"

### apply-actions
- 按钮："提交报名" 替代 "提交报名 Demo"
- 切换按钮："切换账号" / "登录" 二态，去掉 "登录 / 切换账号" 复合文案
- apply-feedback 文案：去掉 "登录后可提交报名；未连接后端时会进入本地 Demo 模式"，改成"先登录或注册账号，就能完成报名。"
