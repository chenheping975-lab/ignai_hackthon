# 后台管理系统规划

版本：v0.5
日期：2026-07-08
状态：规划稿
关联需求：`docs/原始需求_后台管理.md`

## 1. 核心目标

后台管理系统服务一场 IGNAI AI Skillathon 的完整运营闭环：root 初始化后台，管理员配置活动和报名问卷，查看报名数据，使用 AI 初筛和打标签，人工审核，生成有个人归属感的邮件，管理项目提交与展示，最后将作品按赛道、票数、发布时间等规则开放到前台。

后台不是独立 CMS，也不是完整 SaaS。MVP 只服务单场或少量活动的运营管理。

## 2. 入口和角色

| 类型 | 入口 / 标识 | 说明 |
| --- | --- | --- |
| 后台页面入口 | `/root/` | 管理后台前端入口。后续可放在 `frontend/root/` 或独立管理端工程。 |
| root API | `/api/root/*` | root 初始化、root 登录、创始人邮箱绑定、密码重置。 |
| 管理 API | `/api/admin/*` | 活动、用户、报名、项目、邮件、飞书等后台业务接口。 |
| 参赛者 API | `/api/auth/*`、`/api/projects/*` | 参赛者注册登录、报名、项目提交。 |
| 公开 API | `/api/public/*` | 活动页、公开作品、点赞/投票、展示筛选。 |

### 2.1 角色权限

| 角色 | 建议标识 | 权限 |
| --- | --- | --- |
| root / 创始人 | `root` | 首次初始化系统、绑定创始人邮箱、重置 root 密码、创建管理员、查看所有操作日志。 |
| 管理员 | `admin` | 管理活动、报名、问卷、AI 初筛、审核、邮件、项目展示、飞书同步。 |
| 参赛者 | `participant` | 注册登录、报名、提交作品、查看自己的审核和作品状态、点赞作品。 |
| 访客 | 未登录 | 浏览活动页和已公开作品；是否可点赞由活动规则控制，MVP 建议点赞需登录。 |

> 当前数据库 `users.role` 为 `admin/participant`。为贴合本轮需求，建议把角色扩展为 `root/admin/participant`；如果课程实现不想改 enum，也可以临时用 `admin` 加 `system_settings.root_owner_email` 区分 root。

## 3. MVP 模块

| 模块 | 目标 | 主要数据表 |
| --- | --- | --- |
| root 初始化 | 设置 root 密码，绑定创始人邮箱，后续密码重置必须通过该邮箱。 | `users`、`system_settings`、`operation_logs` |
| 仪表盘 | 查看报名数、待审核数、通过数、作品数、点赞数、AI 待处理数。 | `registrations`、`projects`、`votes`、`ai_tasks` |
| 用户管理 | 后台看到用户表，筛选参赛者、管理员、root，禁用异常账号。 | `users` |
| 活动内容管理 | 配置活动标题、时间、报名窗口、提交窗口、评分/点赞开关、赛道。 | `events`、`tracks` |
| 报名问卷管理 | 后台自定义报名问题，支持文本、单选、多选、下拉、文件、URL。 | `form_fields` |
| 报名管理 | 查看每条报名、团队成员、问卷原始字段、AI 标签、审核状态。 | `registrations`、`registration_members` |
| AI 初筛 | 识别乱填、低质量信息、主题方向，生成标签、摘要和建议。 | `ai_tasks`、`registrations.ai_tags_json`、`registrations.ai_summary` |
| 审核与邮件 | 管理员通过/拒绝/候补后生成定制化邮件草稿，可编辑后发送。 | `mail_templates`、`mail_logs` |
| 项目看板 | 查看项目标题、描述、PPT、图片、HTML、MP3、附件，不支持视频。 | `projects`、`project_files` |
| 展示管理 | 管理项目是否公开、排序、赛道分类、票数展示和精选状态。 | `projects`、`votes`、`ratings` |
| 飞书同步 | 报名、项目、附件 metadata、邮件日志同步到飞书协作视图。 | `feishu_sync_logs` |

## 5. 关键业务流

### 5.1 root 初始化

```text
访问 /root/
-> 系统检查是否已设置 root_owner_email
-> 未设置：输入创始人邮箱和 root 密码
-> 后端创建 root 用户或绑定现有 admin 用户
-> 写入 system_settings.root_owner_email
-> 写入 operation_logs
-> 后续只能登录，不能重复初始化
```

### 5.2 报名问卷到审核

```text
管理员配置活动和报名字段
-> 参赛者注册登录
-> 参赛者提交报名和自定义问卷 payload
-> 系统落库 registrations / registration_members
-> AI 任务读取 payload_json
-> 输出 tags、summary、quality_score、spam_risk、recommendation
-> 管理员人工审核通过 / 拒绝 / 候补
-> 生成邮件草稿
-> 管理员编辑确认
-> 发送邮件并写入 mail_logs
```

### 5.3 项目提交到公开展示

```text
管理员设置 submission_deadline 和作品要求
-> 队长提交项目标题、简介、链接、附件
-> 附件类型校验：ppt/pdf/html/zip/mp3/image/doc/other，拒绝 video
-> 后台项目看板审核展示状态
-> 达到公开时间或管理员手动公开
-> 前台作品页按赛道、票数、最新、精选展示
-> 登录用户点赞，重复点赞按 user_id 去重
```

## 6. 业务规则

| 规则 | 说明 |
| --- | --- |
| root 唯一 | MVP 只允许一个 root；root 邮箱写入 `system_settings.root_owner_email`。 |
| 密码安全 | root 和管理员密码只保存 hash，不允许明文；重置需验证创始人邮箱。 |
| 后台权限 | `/api/root/*` 只允许 root；`/api/admin/*` 允许 root/admin；普通参赛者禁止访问。 |
| 自定义字段 | `form_fields.field_key` 在同一活动和 target 下唯一，提交 payload 必须保留原始字段。 |
| AI 初筛 | AI 只输出建议，不自动拒绝报名；完全乱填只能标为高风险，最终由管理员决策。 |
| 邮件发送 | AI 可以生成草稿，但必须人工确认后发送。邮件内容应引用报名者姓名、队伍名、赛道、关键问卷回答。 |
| 报名窗口 | `registration_open=false` 或超过 `registration_deadline` 后禁止提交报名。 |
| 提交窗口 | 超过 `submission_deadline` 后禁止提交或锁定项目，管理员可手动解锁。 |
| 视频限制 | MVP 不支持视频上传、视频外链、视频播放；接口层和后台文案都要明确。 |
| 点赞限制 | MVP 建议登录用户点赞，按 `project_id + voter_user_id` 去重；访客点赞可放 V2。 |
| 展示排序 | 前台支持 `latest`、`votes`、`track`、`featured` 等排序和筛选，后台控制项目是否公开。 |

## 7. 数据契约

| 业务对象 | 主表 | 关键字段 | 说明 |
| --- | --- | --- | --- |
| 用户 / root | `users` | `role`、`email`、`password_hash`、`status` | root/admin/participant 共用用户表。 |
| 系统设置 | `system_settings` | `setting_key`、`setting_value` | 存 `root_owner_email`、初始化状态等。 |
| 活动 | `events` | `registration_open`、`registration_deadline`、`submission_deadline` | 管理报名与作品提交窗口。 |
| 自定义问卷 | `form_fields` | `target_type`、`field_key`、`field_type`、`options_json` | 支持报名和作品两类动态字段。 |
| 报名 | `registrations` | `payload_json`、`ai_tags_json`、`ai_summary`、`status` | payload 保留自定义问卷原文。 |
| AI 任务 | `ai_tasks` | `target_type`、`task_type`、`status`、`result_json` | 标记、摘要、垃圾信息检测、邮件草稿。 |
| 邮件 | `mail_templates`、`mail_logs` | `scene`、`subject_template`、`body_template`、`status` | 草稿和发送记录分离。 |
| 项目 | `projects` | `status`、`track_id`、`title`、`description` | 展示与看板状态流转。 |
| 附件 | `project_files` | `file_type`、`file_path`、`feishu_file_token` | 不允许 video 类型。 |
| 点赞 / 投票 | `votes` | `project_id`、`voter_user_id`、`voter_key` | 前台点赞和后台排序依据。 |

## 9. MVP 与 V2 边界

### V1 必做

- `/root/` 后台入口。
- root 初始化、登录、密码设置和创始人邮箱绑定。
- 用户列表。
- 活动与报名窗口配置。
- 自定义报名问卷。
- 报名列表、详情、AI 初筛结果、人工审核。
- 邮件草稿生成、编辑、确认发送。
- 项目看板、附件查看、展示状态管理。
- 前台作品列表按赛道和票数展示。
- 登录用户点赞。

### V2 可做

- 多活动模板复制。
- 多管理员精细权限。
- 访客匿名点赞风控。
- 飞书 Base / Drive 双向同步。
- 更完整的数据看板和导出。
- 邮件 A/B 模板和批量提醒。

## 10. 待确认

1. 创始人邮箱的真实值是否要写入种子数据，还是只在部署时配置。
2. root 是否必须是独立角色 `root`，还是课程实现中用 `admin` 加配置项表示。
3. 邮件服务优先使用 SMTP、飞书邮箱，还是只先做草稿下载。
4. 点赞是否必须登录；本文档建议 MVP 登录后点赞。
