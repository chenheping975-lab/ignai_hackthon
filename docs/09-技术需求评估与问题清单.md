## 三、技术问题评估

### 问题 1：数据库云端迁移

**现状：**
- 当前数据库在 `118.89.62.40:3306`（远程服务器，非本地）
- `application.properties` 中硬编码了连接信息和密码

**评估：**
- 当前已经是远程数据库，不是本地，所以"迁移到云端"的门槛不高
- 主要风险：密码硬编码在代码中，不安全

**建议方案：**
| 阶段 | 方案 | 说明 |
|------|------|------|
| 开发阶段 | 保持当前远程 MySQL | 够用，不需要改 |
| 上线前 | 环境变量替换硬编码密码 | `spring.datasource.password=${DB_PASSWORD}` |
| 正式部署 | 云数据库 RDS | 阿里云/腾讯云 RDS，自动备份，比自建服务器稳定 |

### 问题 2：文件存储与服务器配置

**现状：**
- 数据库 `project_files` 表有 `file_path` 字段，但 **后端没有实现文件上传接口**
- `application.properties` 没有配置 `multipart` 相关参数
- 当前服务器配置未知（是否有足够磁盘空间存储 PDF/PPT 等大文件）

**评估：**

MySQL **不适合直接存储文件**（BLOB 方式性能差、备份困难）。正确做法是：
- 文件存到**文件系统**（服务器本地目录 或 对象存储）
- 数据库只存**文件路径/URL**（metadata）

**建议方案：**

| 阶段 | 方案 | 成本 | 适合场景 |
|------|------|------|----------|
| MVP | 服务器本地目录 | 0 元 | 开发测试，文件量少 |
| 正式 | 阿里云 OSS / 腾讯云 COS | ~0.1元/GB/月 | 正式上线，文件量大 |

**MVP 实现方式：**
```
用户上传文件 → 后端接收 MultipartFile → 保存到 /uploads/ 目录
→ 文件路径写入 project_files 表 → 前端通过 /uploads/xxx 访问
```

需要在 `application.properties` 添加：
```properties
spring.servlet.multipart.max-file-size=50MB
spring.servlet.multipart.max-request-size=50MB
file.upload-dir=./uploads
```

需要在 `.gitignore` 添加 `uploads/`。
