## 实训授课技术栈

来源文件：[长理软件系统实训授课笔记.docx](/Users/mac/qianzhu%20Vault/project/实训/0702-实训授课资料/长理软件系统实训授课笔记.docx)

- 实训技术：JavaEE + MVC + 手写 SpringMVC / Servlet + MySQL + HTML/CSS。
- 课程项目结构：control / service / dao / model / utils。
- 前端：JSP、HTML、CSS、JS、Bootstrap、jQuery、Toastr。
- 数据库：MySQL8，使用 JDBC 连接。
- 典型功能：注册、登录、session、管理员登录、后台用户审核、用户管理。
- 实训重点：完整业务流程、后台审核、管理员操作、数据库表设计、前后端联调。

结论：黑客松平台可以用 Java 完成实训版。复杂功能应分层：主流程先落本地数据库，AI、飞书、邮件作为可插拔集成。

## HackTour 作品展示参考

参考链接：[https://aihacktour.com/products](https://aihacktour.com/products)

观察到的产品结构：
- 顶部展示赛事状态，历史归档模式下投票和作品提交关闭。
- 导航包括：作品展示、组队广场、打分 / 投票、提交作品、管理系统。
- 作品展示支持按赛道筛选。
- 作品卡片包含：排名、标题、简介、票数、赛道、投票按钮、封面图。
- 未把视频作为核心展示介质，更偏向图片封面、文字简介、链接入口。

对本项目启发：
- 可以做作品展示墙、开放评分和投票，但 MVP 不引入视频能力。
- 平台本地保存封面、PPT、HTML 包、文档和必要附件；视频上传、外链和播放均放到后续扩展。
- 作品展示要为活动结束后的“反复曝光”服务。
