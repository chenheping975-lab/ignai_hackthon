## 目录原则

- `frontend/` 保持轻量，先验证视觉、信息架构和交互。
- `backend/java-web/` 贴合实训课程 Java Web 分层，后续可演进到 Spring Boot。
- `database/` 和 `api/` 是研发同学拆任务的核心依据。
- `docs/` 保留 PRD、设计文档、原始需求和 PDF，方便老师、产品、研发共同查看。
- `uploads/` 只提交 `.gitkeep`，真实附件不进入 Git。
