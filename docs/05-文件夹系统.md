## 目录原则

- `frontend/` 保持轻量，先验证视觉、信息架构和交互；前台按首页、单场活动、单场作品平台拆页。
- `backend/` 以当前 Spring Boot + MyBatis 工程为准，按 Controller / Service / Mapper / POJO 分层。
- `database/` 和 `api/` 是研发同学拆任务的核心依据。
- `docs/` 保留 PRD、设计文档、原始需求和 PDF，方便老师、产品、研发共同查看。
- `/root/` 是后台页面入口规划；后续可放在 `frontend/root/`，也可拆独立管理端。
- `uploads/` 或对象存储只保存真实附件，附件不进入 Git。
