# 黑客松管理平台项目方案总览

版本：v0.4
更新日期：2026-07-08

## 1. 项目定位

IGNAI Hackathon Management Platform 是一个独立的黑客松赛事管理系统，重点服务线下 AI Skillathon / Hackathon 的运营闭环。

核心闭环：

```text
活动配置 -> 报名组队 -> AI 标签 -> 管理员审核 -> 邮件通知
-> 作品提交 -> 开放评分 / 投票 -> 作品展示 -> 飞书备份
```

## 2. 官网与对标网站

| 类型 | 网站 | 用途 | 本项目处理 |
| --- | --- | --- | --- |
| 我们的官网 | https://www.ignai.cn/ | IGNAI 社区首页、品牌、内容展示 | 仅作为友链、品牌资产和首页视觉参考；黑客松平台不强行承接官网文章、成员、记录模块 |
| 本地官网工程 | `/Users/mac/qianzhu Vault/project/IGN AI 官网` | 视觉 token、首页文案、品牌图 | 复用 logo、火炬、深色底、橙黄 + 蓝色信号视觉 |
| 对标网站 | https://aihacktour.com/products | 作品展示、项目卡片、活动曝光、线上互动 | 参考展示结构和曝光路径；MVP 不引入视频能力 |

关键边界：官网和黑客松平台是友链关系，不是同一个内容系统。平台首页可以放官网入口和品牌说明，但不需要把官网已有页面内容全部搬进来。

## 3. MVP 范围

包含：

- 活动首页与后台配置。
- 报名字段配置。
- 个人报名、团队报名、队员信息。
- AI 报名标签、摘要、组队建议。
- 管理员审核通过、拒绝、候补。
- AI 邮件草稿生成，管理员编辑后发送。
- 队伍管理。
- 项目提交和附件上传。
- 作品展示、开放评分、投票互动。
- 本地数据库和文件备份。
- 飞书 Base / 云盘同步记录。

不包含：

- 独立评委角色、评委分配、评委专属评分后台。
- 复杂多轮评审和分级评分。
- 视频上传、视频外链、视频转码、视频在线播放。
- 多租户 SaaS。
- 完整社区官网 CMS。
- AI 自动替代管理员做最终审核。

## 4. 角色

| 角色 | 权限 |
| --- | --- |
| 访客 | 浏览活动页和作品页，按规则参与评分或投票 |
| 参赛者 | 报名、组队、查看审核状态、提交作品、参与开放评分 |
| 队长 | 维护队伍成员、提交团队作品 |
| root / 创始人 | 初始化后台、绑定创始人邮箱、设置 root 密码、创建管理员、查看操作日志 |
| 管理员 | 配置活动、审核报名、管理队伍、邮件发送、作品管理、飞书同步、评分规则配置 |

## 5. 前端视觉方向

前端使用 IGNAI 官网的品牌气质：

- 深色背景。
- 橙黄火炬感作为主强调色。
- 蓝色作为信号、连接和同步状态辅助色。
- 首页第一屏直接呈现黑客松运营工作台，不做单纯宣传页。
- 顶部和页脚保留 IGNAI 官网、AI Hack Tour 对标站链接。

## 6. 技术路线

实训版当前以后端合入的 Spring Boot 工程为准：

- 前端：HTML / CSS / JavaScript 静态原型，后续可拆 JSP 或接入 Vue / React。
- 后端：Spring Boot 3 / MyBatis / Java 17。
- 数据库：MySQL 8。
- 文件：本地文件备份，数据库保存 metadata。
- AI：独立 service 封装，先 mock，后接 OpenAI / 其他模型。
- 飞书：独立 integration service，先 CSV / JSON 导出，后接 Base / Drive API。
- 邮件：独立 mail service，发送前必须人工确认。

## 7. 交付物

- `frontend/`：可直接预览的前端原型。
- `backend/`：Spring Boot + MyBatis 后端工程。
- `database/schema.sql`：MySQL 建表脚本。
- `api/openapi.yaml`：接口定义。
- `docs/`：PRD、概要设计、详细设计、业务逻辑、研发 PDF。
