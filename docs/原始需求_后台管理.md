## 1. 原始输入

这个目前又合并了一个后端的端口，但是顺着我们的业务逻辑来说，我们目前缺少一个后台的管理员系统，后台管理员的系统的功能，包括一个新的/资源比如/root 后缀，其次，root 的密码需要设置，且绑定了创始人的邮箱，更换注册密码指定邮箱，然后呢，登录后台管理系统，需要能够看到用户表，其次，我们里面有个黑客松报名表，我们会设置，黑客松报名的问题，所以，后台要看到一个一个的报名情况，也就是报名的信息，然后我们的问题可能是自定义设计的，所以，我们后台要有一个黑客松的内容管理平台，然后参赛者，注册登录之后是可以完成参赛的，之后报名的所以问卷信息沉淀起来的话，我们有个 ai 自动读报名信息，并且归类，打上 tag 或者自然语义分析，首先要过滤一轮，完全乱填信息的，之后，我们可以批准通过之后，要可以定制化的定制发送邮件信息的内容，要和报名者的填写信息有关，这样才有归属感，好的，后台注册之外，要可以设置报名提交的窗口和时间，和要求，这个也是定制化的，提交完成之后，要有看板可以看项目，有 ppt,文字描述，图片等信息，不涉及视频，存在后台管理平台中，之后，用户端再开放完成之后，可以预览，并且注册登录的用户可以选择点赞作品，然后后台可以管理这些项目的展示，并且，可以根据票数，以及赛道等分类咋前端展示，大概是这些功能，先归类我的原始需求_后台管理为 doc 的 markdown文档，其次，开始规划 API 设计，规划好我们的文档结构每次完成对话之前，commit 本轮涉及修改的文件并 push

## 2. 需求归类

### 2.1 后台入口与账号体系

| 需求点 | 详细说明 | 当前处理 |
|--------|----------|----------|
| 后台入口 | 后台管理员系统，入口为 `/root` 后缀，与前台分离 | 前端 `frontend/root/` 已有 MVP 页面 |
| root 密码初始化 | 首次使用需设置 root 密码，只允许执行一次 | 规划 `POST /api/root/bootstrap` |
| 创始人邮箱绑定 | root 密码绑定创始人邮箱，更换密码需验证指定邮箱 | 规划存储于 `system_settings` 表 |
| root 登录 | root 账号独立登录后台管理系统 | 规划 `POST /api/root/login`，独立 JWT |
| 密码修改 | 修改 root 密码需二次验证 | 规划 `PATCH /api/root/password` |

### 2.2 用户管理

| 需求点 | 详细说明 | 当前处理 |
|--------|----------|----------|
| 用户列表 | 登录后台后能看到用户表 | 规划 `GET /api/admin/users` |
| 用户筛选 | 支持角色、状态、关键词筛选 | 规划查询参数 |
| 用户详情 | 查看单个用户的报名、队伍、项目摘要 | 规划 `GET /api/admin/users/{userId}` |
| 用户状态管理 | 启用/禁用用户 | 规划 `PATCH /api/admin/users/{userId}/status` |
| 创建管理员 | root 可创建管理员账号 | 规划 `POST /api/admin/users/admins` |

### 2.3 活动与问卷管理

| 需求点 | 详细说明 | 当前处理 |
|--------|----------|----------|
| 活动管理 | 创建、编辑黑客松活动 | 规划 `POST/PUT /api/admin/events` |
| 报名窗口设置 | 设置报名提交的窗口、时间、要求，定制化 | 复用 `events.registration_open`、`registration_deadline` |
| 提交窗口设置 | 设置项目提交的截止时间 | 复用 `events.submission_deadline` |
| 自定义问卷 | 报名问题由后台自定义设计（字段类型、排序、必填） | 复用 `form_fields` 表 |
| 赛道管理 | 创建和管理赛事赛道 | 规划 `POST /api/admin/events/{eventId}/tracks` |

### 2.4 报名管理与 AI 初筛

| 需求点 | 详细说明 | 当前处理 |
|--------|----------|----------|
| 报名列表 | 后台看到每一条黑客松报名信息 | 规划 `GET /api/admin/registrations` |
| 报名详情 | 查看报名的完整问卷回答、成员信息 | 规划 `GET /api/admin/registrations/{id}` |
| AI 自动归类 | AI 读取报名信息，自动归类、打 tag、自然语义分析 | 复用 `ai_tasks`、`registrations.ai_tags_json` |
| AI 乱填过滤 | AI 首先过滤完全乱填的信息，识别 spam 风险 | 规划 `spamRisk` 字段 |
| 人工审核 | 管理员批准通过/拒绝/候补 | 规划 `PATCH /api/admin/registrations/{id}/review` |
| 批量审核 | 支持批量操作 | 规划 `POST /api/admin/registrations/batch-review` |

### 2.5 邮件 Agent

| 需求点 | 详细说明 | 当前处理 |
|--------|----------|----------|
| 邮件模板 | 管理邮件模板（批准、拒绝、提醒等场景） | 复用 `mail_templates` 表 |
| 个性化邮件草稿 | 批准通过后，基于报名者填写信息生成定制化邮件内容 | 规划 `POST /api/admin/registrations/{id}/mail-draft` |
| 人工确认发送 | 邮件草稿允许人工编辑，确认后才发送 | 规划 `POST /api/admin/mail/send` |
| 发送记录 | 查看邮件发送日志 | 复用 `mail_logs` 表 |

### 2.6 项目看板与公开展示

| 需求点 | 详细说明 | 当前处理 |
|--------|----------|----------|
| 后台项目看板 | 提交完成后，后台看板查看项目，包含 PPT、文字描述、图片等，不涉及视频 | 规划 `GET /api/admin/projects` |
| 项目详情 | 查看项目的完整信息和附件列表 | 规划 `GET /api/admin/projects/{projectId}` |
| 展示管理 | 后台管理项目的公开/隐藏、精选、排序 | 规划 `PATCH /api/admin/projects/{projectId}/showcase` |
| 前台预览 | 用户端开放后可预览作品 | 复用 `GET /api/public/projects` |
| 点赞功能 | 注册登录用户可以点赞作品 | 复用 `votes` 表 |
| 分类展示 | 根据票数、赛道等分类在前端展示 | 规划排序和筛选参数 |

### 2.7 MVP 限制

| 限制项 | 说明 |
|--------|------|
| 不支持视频 | MVP 阶段不支持视频上传，返回 `VIDEO_NOT_SUPPORTED_IN_MVP` |
| 文件类型 | 仅支持 PPT、PDF、HTML、ZIP、MP3、图片、文档 |

## 3. 核心业务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        后台管理员流程                            │
├─────────────────────────────────────────────────────────────────┤
│  root 初始化 → 登录后台 → 创建活动 → 设计报名问卷 → 设置窗口时间  │
│       ↓                                                         │
│  查看报名列表 → AI 初筛（打标签/过滤乱填） → 人工审核             │
│       ↓                                                         │
│  生成个性化邮件 → 确认发送通知                                   │
│       ↓                                                         │
│  查看项目看板 → 管理展示状态 → 设置精选/排序                     │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                        参赛者流程                               │
├─────────────────────────────────────────────────────────────────┤
│  注册/登录 → 查看活动 → 提交报名（填写自定义问卷）               │
│       ↓                                                         │
│  等待审核 → 收到通知邮件                                        │
│       ↓                                                         │
│  创建队伍 → 提交项目（PPT/文字/图片） → 正式提交                 │
│       ↓                                                         │
│  浏览作品 → 点赞投票                                            │
└─────────────────────────────────────────────────────────────────┘
```
