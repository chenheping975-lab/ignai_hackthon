# 后台管理与赛事预览 UI 梳理

版本：v0.7
日期：2026-07-08
状态：静态 MVP 已迭代
关联文档：`docs/07-后台管理系统规划.md`、`docs/design/产品边界与视觉方向.md`

## 1. 为什么要拆成两个 UI

当前平台有两个尚未完全梳理的 UI 面：

| UI 面 | 主要用户 | 核心问题 | 设计气质 |
| --- | --- | --- | --- |
| 后台管理 UI | root、管理员、运营同学 | 怎么配置活动、看报名、做 AI 初筛、审核、发邮件、管理作品展示 | 克制、密集、可扫描、少装饰 |
| 整体赛事预览 UI | 访客、参赛者、队长、管理员预览 | 一眼看清赛事状态、自己下一步该做什么、作品如何被展示 | IGNAI 品牌感、火炬橙、活动现场感、状态清晰 |

后台管理 UI 是“操作系统”。整体赛事预览 UI 是“活动现场和作品橱窗”。两者可以共享品牌色和组件语言，但不能共享同一种页面密度。

## 2. 后台管理 UI

### 2.1 页面地图

```text
/root/
  00 初始化 / 登录
  01 概览 Dashboard
  02 用户管理 Users
  03 活动管理 Event CMS
     - 基础信息
     - 时间窗口
     - 赛道
     - 报名问卷
     - 作品提交要求
     - 前台预览
  04 报名管理 Registrations
     - 列表
     - 详情抽屉
     - AI 初筛
     - 审核
     - 邮件草稿
  05 项目看板 Projects
     - 列表 / 看板
     - 附件
     - 展示状态
     - 票数 / 赛道排序
  06 邮件中心 Mail
  07 飞书同步 Feishu
  08 操作日志 Logs
```

### 2.2 总体布局

后台采用固定管理端 Shell：

```text
┌──────────────────────────────────────────────────────────────┐
│ Top Bar: 当前活动 ▾  全局搜索  同步状态  当前账号             │
├──────────────┬───────────────────────────────────────────────┤
│ Side Nav     │ Page Header                                    │
│ 概览         │ 标题 / 描述 / 主操作按钮 / 当前状态             │
│ 用户         ├───────────────────────────────────────────────┤
│ 活动         │ Filters / Tabs / Saved Views                   │
│ 报名         ├───────────────────────────────────────────────┤
│ 项目         │ Main Workspace                                 │
│ 邮件         │ 表格 / 看板 / 表单 / 详情抽屉                  │
│ 飞书         │                                               │
│ 日志         │                                               │
└──────────────┴───────────────────────────────────────────────┘
```

布局原则：

- 左侧导航稳定，后台用户不用每页重新学习。
- 顶部保留活动切换器，因为后续会有多场活动。
- 主工作区优先表格、筛选、状态、批量操作。
- 详情不要频繁跳页，优先右侧抽屉或 split view。
- 不使用大 hero、营销式卡片堆叠、夸张装饰。

### 2.3 核心页面原型

#### A. root 初始化 / 登录

目的：第一次部署时设置 root 和创始人邮箱；之后只显示登录。

```text
┌──────────────────────────────┐
│ IGNAI Skillathon Admin       │
│ Root 初始化                  │
├──────────────────────────────┤
│ 创始人邮箱  [              ] │
│ Root 密码   [              ] │
│ 确认密码    [              ] │
│                              │
│ [ 初始化后台 ]               │
└──────────────────────────────┘
```

状态：

- 未初始化：显示 root 初始化。
- 已初始化：显示登录。
- 邮箱已绑定：重置密码必须走创始人邮箱验证。

#### B. 概览 Dashboard

目的：管理员进入后台后一眼看到赛事运行状态。

```text
┌ 概览：IGNAI AI Skillathon ────────────── [前台预览] [新建活动] ┐
│ 报名 128  待审核 34  已通过 76  作品 18  点赞 624  AI 待处理 9 │
├───────────────────────────┬──────────────────────────────────┤
│ 今日待处理                │ 赛事时间轴                       │
│ - 12 条报名待审核         │ 报名开放 ━━━ 当前 ━━━ 提交截止    │
│ - 4 封邮件草稿待确认      │                                  │
│ - 3 个项目待公开          │                                  │
├───────────────────────────┴──────────────────────────────────┤
│ 最近报名 / 最近项目 / 飞书同步异常                            │
└───────────────────────────────────────────────────────────────┘
```

关键组件：

- 指标卡：报名、审核、作品、点赞、AI 任务。
- 时间轴：报名窗口、提交窗口、评分窗口。
- 待办列表：可直接跳到对应模块。
- 异常提醒：飞书同步失败、邮件发送失败、AI 失败。

#### C. 活动管理 / 问卷编辑

目的：配置赛事内容和报名问题。这里是“黑客松内容管理平台”的主体。

```text
┌ 活动管理 ─────────────────────────── [保存] [预览赛事页] ┐
│ Tabs: 基础信息 | 时间窗口 | 赛道 | 报名问卷 | 作品提交要求 │
├──────────────────────┬─────────────────────────────────────┤
│ 字段列表              │ 字段编辑                            │
│ 01 姓名   required    │ Label       [ 你想做什么项目？     ] │
│ 02 手机   required    │ Type        [ textarea ▾          ] │
│ 03 赛道   select      │ Required    [x]                    │
│ 04 项目方向 textarea  │ Placeholder [ 简单描述问题和想法 ] │
│ [+ 新增字段]          │ Options     [ ... ]                │
└──────────────────────┴─────────────────────────────────────┘
```

交互规则：

- 字段支持拖拽排序。
- 保存前校验 `field_key` 唯一。
- 问卷编辑右侧实时显示“参赛者看到的表单预览”。
- 作品提交要求和报名问卷分开，避免选手报名阶段被项目信息压垮。

#### D. 报名管理 / AI 初筛 / 审核

目的：把报名信息沉淀、AI 归类、人工审核和邮件通知串成一屏。

```text
┌ 报名管理 ───────────────────────────────────────────────┐
│ 状态: 全部 / 待审核 / 已通过 / 候补 / 已拒绝             │
│ 筛选: 赛道 ▾  AI 标签 ▾  风险 ▾  关键词 [           ]    │
├───────────────┬────────┬──────────┬────────┬───────────┤
│ 姓名/队伍      │ 赛道   │ AI 标签   │ 风险   │ 状态/操作  │
│ Torch Lab      │ Agent  │ 全栈,工具 │ low    │ 待审核     │
│ ...           │ ...    │ ...      │ ...    │ [详情]     │
└───────────────┴────────┴──────────┴────────┴───────────┘

右侧详情抽屉：
┌ 报名详情：Torch Lab ───────────────────────────────────┐
│ 基础信息 / 问卷原文 / 队员 / AI 初筛 / 邮件             │
│ AI 摘要：团队希望做一个面向实训课程的 Agent 工具。       │
│ 标签：Agent / 教育 / 全栈                               │
│ 风险：低                                                │
│ [通过并生成邮件] [候补] [拒绝]                           │
└───────────────────────────────────────────────────────┘
```

交互规则：

- AI 初筛只给建议，不能自动拒绝。
- 审核动作必须写入操作日志。
- 通过 / 候补 / 拒绝后进入邮件草稿，不直接发送。
- 邮件草稿要引用报名者姓名、队伍名、赛道和关键回答。

#### E. 项目看板 / 展示管理

目的：管理提交作品、附件和前台展示顺序。

```text
┌ 项目看板 ───────────────────────────── [公开展示预览] ┐
│ Tabs: 全部 / 草稿 / 已提交 / 已公开 / 已归档             │
│ 筛选: 赛道 ▾  排序: 票数 ▾  关键词 [                ]   │
├───────────────────────────────────────────────────────┤
│ Project Card                                           │
│ Campus Agent Hub                                       │
│ Agent Tools · 128 likes · PPT 1 · Image 2 · HTML 1      │
│ 状态: 已提交  [查看附件] [设为公开] [精选]              │
└───────────────────────────────────────────────────────┘
```

附件规则：

- 支持 PPT、PDF、HTML、ZIP、MP3、图片、文档。
- 不支持视频上传、视频外链和在线播放。
- 附件在后台看板可查看 metadata，原件下载走受控接口。

## 3. 整体赛事预览 UI

### 3.1 它解决什么问题

整体赛事预览 UI 不是单纯首页，也不是后台表单。首页只负责活动精神和活动入口；单场活动页与作品平台要回答三个问题：

1. 这场黑客松现在处于什么阶段？
2. 我作为访客 / 参赛者 / 队长下一步该做什么？
3. 作品如何被看到、点赞、按赛道展示？

建议把它整理为一套“状态化赛事页”，由后台配置驱动，前台和后台预览共用同一套结构。

v0.7 已把前台拆成三层：

| 层级 | 页面 | 主要职责 |
| --- | --- | --- |
| 社区活动首页 | `/` | 传达 IGNAI 活动精神，列出可报名黑客松活动。 |
| 单场活动页 | `/event.html?id=...` | 展示活动状态、赛道、报名表单、提交要求和作品平台入口。 |
| 单场作品平台 | `/projects.html?event=...` | 展示当前活动作品，支持赛道筛选、排序和点赞。 |

### 3.2 页面地图

```text
赛事预览 / public event
  Hero: 活动名、状态、地点、时间、主行动
  Status Rail: 报名开放 / 组队中 / 提交中 / 展示中 / 已归档
  Tracks: 赛道和作品方向
  Apply Preview: 报名入口和报名问题预览
  Participant Console: 登录后的个人状态
  Project Submission Preview: 作品提交要求
  Showcase: 作品列表、赛道筛选、点赞排序
  Archive: 赛后回看和内容沉淀
```

### 3.3 状态化布局

```text
┌ Hero ───────────────────────────────────────────────────┐
│ IGNAI AI Skillathon                                     │
│ 长沙 · 两天一夜 · 当前：开放报名                         │
│ [提交报名] [查看作品]                                    │
├ 状态条 ─────────────────────────────────────────────────┤
│ 报名开放 ●━━━━ 组队中 ○━━━━ 提交中 ○━━━━ 展示中 ○        │
├ 我的状态 / 下一步 ───────────────────────────────────────┤
│ 游客：登录后可报名                                       │
│ 已报名：等待审核 / 审核通过 / 进入提交                   │
├ 赛道与问题 ─────────────────────────────────────────────┤
│ Agent 工具 | 教育实训 | 内容生产 | 商业应用              │
├ 作品提交要求 ───────────────────────────────────────────┤
│ 标题、简介、Demo 链接、代码链接、PPT/HTML/MP3/图片       │
├ 作品展示 ───────────────────────────────────────────────┤
│ 筛选：全部 / Agent / 教育 / 内容 / 商业  排序：票数 / 最新│

```

### 3.4 用户状态

| 用户状态 | 页面要显示什么 | 主行动 |
| --- | --- | --- |
| 访客 | 活动信息、赛道、报名入口、公开作品 | 登录 / 注册 / 查看作品 |
| 已登录未报名 | 报名表入口、问卷预览、截止时间 | 提交报名 |
| 已报名待审核 | 报名摘要、AI 标签可隐藏、审核进度 | 查看报名状态 |
| 已通过参赛者 | 组队信息、现场安排、作品提交倒计时 | 完善队伍 / 提交作品 |
| 队长 | 队员、项目草稿、附件状态 | 提交作品 |
| 已提交 | 项目状态、是否公开、评分/点赞入口 | 查看展示页 |
| 赛后访客 | 作品归档、赛道筛选、热门作品 | 浏览 / 点赞 / 分享 |

### 3.5 后台预览模式

后台活动管理里需要一个“预览赛事页”按钮，支持切换身份：

```text
预览赛事页
┌─────────────────────────────────────────────┐
│ 预览身份: 访客 ▾  设备: Desktop / Mobile     │
│ 阶段: 报名开放 / 提交中 / 展示中              │
├─────────────────────────────────────────────┤
│ iframe / preview canvas                      │
│ 显示前台赛事页                               │
└─────────────────────────────────────────────┘
```

预览身份：

- 访客
- 已登录未报名
- 已报名待审核
- 已通过参赛者
- 队长
- 已提交项目

预览用途：

- 管理员确认报名窗口、文案、问卷字段是否正确。
- 管理员确认提交窗口和项目要求是否清楚。
- 管理员确认公开作品展示排序是否符合预期。

### 3.6 作品展示 UI

作品展示不做视频，重点做“可扫描 + 可传播”：

```text
┌ 作品展示 ───────────────────────────────────────────────┐
│ 筛选: 全部 / Agent / 教育 / 内容 / 商业                  │
│ 排序: 精选 / 票数 / 最新 / 评分                          │
├────────────────────────────────────────────────────────┤
│ [封面图] Campus Agent Hub                               │
│ Agent Tools                                             │
│ 面向高校实训的课程 Agent 与作品沉淀平台。                │
│ PPT · HTML · GitHub · Demo                              │
│ 128 likes   4.7 score   [点赞] [查看详情]                │
└────────────────────────────────────────────────────────┘
```

项目卡片字段：

| 字段 | 来源 |
| --- | --- |
| 封面 | `project_files` 或默认 IGNAI 品牌图 |
| 标题 | `projects.title` |
| 一句话 | `projects.tagline` |
| 赛道 | `tracks.name` |
| 描述 | `projects.description` |
| 链接 | `demo_url`、`code_url` |
| 附件标签 | `project_files.file_type` |
| 点赞 | `votes` |
| 评分 | `ratings` |

## 4. 两套 UI 的关系

| 关系 | 说明 |
| --- | --- |
| 同一活动数据 | 后台配置 `events`、`tracks`、`form_fields`，赛事预览读取这些配置。 |
| 同一作品数据 | 后台管理 `projects`、`project_files`，前台只展示 published 项目。 |
| 同一视觉基因 | 都使用 IGNAI 火炬橙和暖纸底，但后台更克制、密集。 |
| 不同页面密度 | 后台信息密度高；赛事预览更叙事、更适合移动端传播。 |
| 后台可预览前台 | 活动管理页提供 preview mode，避免配置和前台效果割裂。 |

## 5. 推荐落地顺序

### 第一步：静态 UI 原型

1. 新建 `frontend/root/` 管理后台静态原型。
2. 新建后台 Shell、概览、活动管理、报名管理、项目看板。
3. 把当前首页的 `#apply` 区域整理成“赛事预览控制台”。
4. 增加赛事预览身份切换假数据。

### 第二步：接入真实 API

1. root 初始化和登录。
2. 活动配置和问卷字段读取。
3. 报名列表和详情。
4. 项目看板和公开作品列表。
5. 点赞。

### 第三步：补交互细节

1. 报名详情抽屉。
2. AI 初筛面板。
3. 邮件草稿编辑器。
4. 项目附件预览。
5. 后台内嵌前台预览。

## 6. UI 验收标准

| UI | 验收标准 |
| --- | --- |
| 后台管理 UI | 管理员能在 3 次点击内找到待审核报名、活动窗口、项目公开状态。 |
| 报名管理 | 能看到原始问卷、AI 摘要、风险、审核按钮和邮件草稿入口。 |
| 问卷编辑 | 能新增字段，编辑标题、类型、必填、占位提示和选项，并预览参赛者表单。 |
| 项目看板 | 能按状态、赛道、票数筛选，并明确哪些项目已公开。 |
| 赛事预览 UI | 访客和参赛者能从首页进入单场活动，并一眼知道当前阶段和下一步行动。 |
| 作品展示 | 能按赛道和票数排序，登录用户能点赞，视频能力不出现。 |
| 移动端 | 赛事预览页必须优先适配移动端；后台管理移动端只需可查阅，复杂操作优先桌面。 |
