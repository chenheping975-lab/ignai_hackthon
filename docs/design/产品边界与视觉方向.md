# 产品边界与视觉方向

## 产品边界

黑客松平台是独立赛事管理系统，不是 IGNAI 官网的子模块集合。

与官网的关系：
- 导航或页脚可放 `IGNAI 官网` 友链。
- 官网可反向放 `黑客松平台` 入口。
- 可复用 IGNAI Logo、火炬符号、配色和视觉气质。
- 不强制接入官网的成员、文章、社区记录、活动列表等内容栏目。

## 平台核心场景

1. 管理员创建和配置一场黑客松活动。
2. 选手报名，选择个人 / 团队，填写技能、赛道、组队需求。
3. 系统用 AI 整理报名信息、打标签、给管理员审核建议。
4. 管理员审核报名和组队结果，批量发送邮件通知。
5. 队伍在比赛期间提交项目资料和作品。
6. 大家按活动规则参与开放评分，选手或观众参与投票互动。
7. 活动结束后形成作品展示页，继续传播和回看。
8. 数据和附件在本地保留一份，同时同步 / 备份到飞书。

## 视觉方向

建议方向：`暗色科技底 + 火炬橙强调 + 少量蓝色状态色`。

色彩：
- 背景：`#07080C` / `#0D0E14`
- 主强调：`#FF7A18`
- 辅助强调：`#5DA9FF`
- 成功 / 开放：可使用偏青绿色
- 警示 / 截止：使用橙红色

页面气质：
- 公共展示页可以更强视觉冲击，类似 HackTour 的赛事感。
- 管理后台要更克制，优先清晰、密集、可扫描。
- 不做大段官网式社区介绍，首页直接进入赛事信息、报名入口、作品展示和友链。

## 关键页面

公共侧：
- 赛事首页
- 报名页
- 组队广场
- 作品提交页
- 作品展示页
- 作品详情页

管理侧：
- 管理登录
- 活动配置
- 报名表单配置
- 报名审核
- 队伍管理
- 作品管理
- 评审管理
- 邮件通知
- 飞书同步记录

评分侧：
- 开放评分入口
- 评分口令 / 来源限制
- 评分结果统计
- 投票互动

## 文件与视频策略

MVP 不引入视频能力。

建议支持：
- PPT / PDF / 文档
- HTML 或 ZIP 项目包
- 音频 MP3
- 图片封面
- GitHub / Demo 链接

谨慎支持：
- MP4 本地上传、视频外链、视频在线播放都放到后续扩展。

原因：
- 视频存储占用大。
- 上传失败和转码复杂。
- 在线预览成本高。
- 竞品展示页也主要采用封面图、文字和投票，不依赖视频。
