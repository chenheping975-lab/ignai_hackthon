# TODO：前后台动态联动 MVP

版本：v0.1
日期：2026-07-08
任务体量：非简单任务，中等改造

## 目标

打通“后台管理系统配置活动内容 → 后端写入云端 MySQL → 前台活动详情页动态读取展示”的 MVP 链路。

本轮只做最小可验证闭环：

1. 后台可以读取当前活动配置。
2. 后台可以编辑活动基础信息、赛道、报名问题。
3. 后端把配置写入 `events`、`tracks`、`form_fields`。
4. 前台活动详情页读取同一份活动配置。
5. 构建、后端编译、API smoke test 通过。

## 当前判断

这个链路不是服务端渲染模板方案，而是更适合本项目现状的“Vue SPA + Spring Boot API + MySQL”方案。后台管理系统负责写数据库，前台页面运行时请求后端公开接口读取数据库内容。后续如果要做 SEO 或静态快照，可以再追加 SSR/SSG，不影响本轮 MVP。
