## 上一轮（v0.8）回顾

v0.8 已经把首页精简为「精神 + 活动列表」，活动卡片只留一个「进入活动」按钮。但实测后发现**信息密度问题没解决**：

1. **首页活动列表整段堆在 Hero 下方**，访客一进来既要看精神、又要看 SignalStrip、又要看 3 场活动卡片，注意力被切成三段。
2. **活动详情页直接把报名表单怼在脸上**：Hero + 状态轨道 + 赛道 + **报名表单** + 提交要求全部挤在一个页面，分不清「现在在浏览」还是「现在在报名」。
3. **缺少独立的报名编辑页**：报名动作和活动说明没有页面层级隔离，导致「加入比赛」这个关键决策点被弱化。
4. **缺少「我的报名」入口**：用户报名后没办法回看自己的状态，闭环不完整。

## 本轮目标

参考 aihacktour.com 的「静态展示 → 决策点 → 动作页」三级漏斗，把页面层级彻底拆开：

```
落地页（静态品牌）→ 活动列表 → 活动详情（决策）→ 报名编辑页（动作）→ 我的报名（闭环）
```

**每页只干一件事，按钮意图单一。**

## 目标页面层级

```
/                        落地页（纯静态品牌 + 精选活动）
/events                  活动列表（所有比赛一览 + 状态筛选）
/event/:id               活动详情（Hero + 状态轨道 + 赛道 + 提交要求）
/event/:id/join          报名编辑页（独立全屏表单，登录后才能进）
/my/registrations        我的报名（查看状态）
/projects                作品展示
/login                   登录（支持 ?redirect= 回跳）
/register                注册
/root                    后台管理
```

## 验收口径

- 首页 `/` 不再出现完整的活动列表，只有品牌信息 + 1-2 个 Featured 精选；主 CTA 指向 `/events`。
- 活动详情页 `/event/:id` **不再有报名表单**；报名通过「加入比赛」按钮跳转到独立的 `/event/:id/join`。
- 报名编辑页 `/event/:id/join` 是独立路由，未登录访问会自动跳到登录页并带 `redirect` 参数；登录后自动回跳。
- 提交报名成功后跳到 `/my/registrations`，能看到自己的报名记录（或接口未就绪时的占位提示）。
- 导航栏不再有失效的锚点（`#events`、`#spirit`）；登录后多出「我的报名」入口。
- 整条链路浏览器走通：首页 → 活动列表 → 活动详情 → 加入比赛 → 登录 → 报名 → 我的报名，无 console error。

## v1.0 新增：首页重做（讲故事结构）

### 用户反馈

> 首页的 UI 做得一坨。Hero「连接创造者」字体没有居中、突兀；首页要讲一个故事，但当前静态故事没讲好。
> 参考 aihacktour 的**逻辑结构**（不要照搬其具体内容）。

### 根因诊断

- 全局 `.hero-inner` 是「左文 + 右图」双列网格，但 `HomeView.vue` 用了自己的 `.hero-content` 单列布局；
- 同时全局 `.hero::before`（IGNAI 巨型水印）和 `.hero::after`（橙色椭圆边框）还在渲染；
- 结果：文字左挤、右半空、视觉断层，加上水印干扰 → 突兀。

### aihacktour 叙事结构（参考，内容用 IGNAI 自己的）

1. 开场引言（大佬话定调）
2. 黑客松介绍（散文式描述）
3. 关键时间点与流程（⚠️ 截止提醒 + 日程）
4. 谁会在这里相遇（3 类参与者，01/02/03 编号）
5. 主办方介绍
6. 收尾引言 + CTA

### 新首页结构（IGNAI 版）

```
1. Hero（左文右图，对齐全局 .hero-inner）
   ├─ 左：eyebrow / 大标题 / 副标题 / 双 CTA
```
