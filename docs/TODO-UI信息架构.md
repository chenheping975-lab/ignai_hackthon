## 上一轮（v0.3）回顾

v0.3 已经把链路拆成「首页 → 单场活动 → 单场作品平台」三层，但实测后用户反馈四个问题：

1. **首页 reveal 动画让滚动有阻尼感**：滚动时所有 section 逐个 fade-in，加上 0.6~0.8s 的 expo 缓动，整页感觉粘滞。
2. **首页活动卡片同时给「进入活动」和「作品平台」两个按钮**，破坏了「先选比赛 → 进入比赛 → 才看到作品平台」的链路。
3. **首页 `#works-entry` 栏目在硬讲流程**，和「活动精神传达 + 活动列表」的定位冲突。
4. **后台报名问题编辑器布局挤**：标题、题型、占位、必填、选项挤在一行；切换题型会全量重渲染 DOM 导致正在编辑的输入框失焦；新增问题之后看不清怎么编辑。

## 本轮目标

把上面 4 个问题全部落成可验收的改动，并保证整条 mock 链路（首页 → 活动详情 → 报名 → 作品平台 → 后台审核）能在浏览器走完一遍。

参考站点：[aihacktour.com](https://aihacktour.com/) — 已抓取。

参考结论：
- 首页结构：Hero 文案 + 当前场活动信息卡 + 议程时间线 + 参与人群 + 主办方/合作方。
- **没有独立的「行动流程」栏目**；议程是单场活动信息卡的一部分。
- **首页不直接展示作品/项目入口**；入口按钮只有报名/提交类。
- 首页是一场活动为主视觉，不堆流程说明。

IGNAI 场景是多场赛事聚合，所以保留「列表 → 详情」结构，但**借鉴 aihacktour 把首页精简为精神 + 活动列表**的思路。

## 验收口径

- 首页快速向下滚不再有「粘滞」感；reveal 入场动画 ≤0.3s。
- 首页只有「活动精神」+「活动列表」两类内容；不再出现 `#works-entry` 或「作品平台」二级入口。
- 活动卡片只有一个 CTA「进入活动」；进入活动详情页后才看到「报名」和「作品平台」。
- 后台「活动管理 → 新增问题」之后：标题、题型、必填、占位、选项（仅当题型为下拉/单选/多选）全部可在原位编辑；连续输入不会失焦。
- 整条 mock 链路：首页 → 活动详情 → 报名 → 后台审核 → 作品平台，能完整跑通且不报 console error。
