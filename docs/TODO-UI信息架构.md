## MVP 覆盖交互

- root 初始化：填写创始人邮箱和 root 密码，更新 root 状态。
- 概览：查看报名、待审核、作品、点赞、AI 待处理等指标。
- 活动管理：新增报名问题并同步参赛者表单预览。
- 报名管理：筛选报名，打开详情抽屉，运行 AI 初筛，审核通过并生成邮件草稿。
- 项目看板：按状态筛选，项目公开 / 精选切换。
- 赛事预览：按身份和赛事阶段切换，预览下一步行动和作品展示。

| 问题 | 当前建议 |

| “整体赛事预览 UI”是否是公开页面还是后台预览？ | 建议两者合并设计：公开赛事页是用户看到的页面，后台提供“以访客/参赛者/已通过队长身份预览”的模式。 |
| `/root/` 后台 UI 是否先做静态原型？ | 建议先做静态原型，采用左侧导航 + 顶部活动切换 + 主工作区 + 详情抽屉。 |
| 赛事预览是否复用当前首页？ | 建议复用当前首页视觉，但从“宣传叙事”整理成“赛事状态预览 + 当前行动入口 + 作品曝光”。 |
