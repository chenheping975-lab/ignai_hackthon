# 需求审核与迭代建议

## 已确认边界

- 黑客松平台是独立系统。
- 官网只做友链，不把社区内容模块并入。
- 可以复用 IGNAI 品牌视觉，但产品结构以赛事管理为主。
- Java 实训版可行，先做完整流程，再做复杂集成。

## 核心功能拆解

### 1. 活动配置

管理员配置活动基本信息、赛道、奖项、报名截止时间、提交截止时间、开放评分规则。

这是平台的根，因为后续报名、作品、评审都挂在同一场活动上。

### 2. 报名与组队

支持个人报名、团队报名、现场组队需求。

关键规则：
- 同一个参赛者不能在同一活动中加入多个队伍。
- 团队报名要有队长。
- 报名需要审核，通过后才正式进入名单。

### 3. AI 辅助审核

AI 负责：
- 提取技能标签。
- 提取赛道倾向。
- 总结报名信息。
- 给组队 / 审核建议。

AI 不负责：
- 自动通过 / 拒绝报名。
- 自动替代用户做最终评分。
- 自动发送邮件。

### 4. 邮件通知 Agent

系统根据报名状态、活动信息、队伍信息生成邮件草稿。

管理员必须可以：
- 查看草稿。
- 修改内容。
- 批量发送。
- 查看发送记录。

### 5. 项目提交

支持团队提交作品资料和附件。

MVP 推荐：
- PPT / PDF / HTML / ZIP / MP3 / 图片。
- Demo 链接、GitHub 链接。

不建议 MVP：
- 大 MP4 本地上传。
- 视频外链字段。
- 视频转码。
- 在线视频播放器。

### 6. 开放评分与投票

撤掉“评委”独立角色，不做评委账号、评委分配和评委专属评分页。

- 开放评分：所有人按活动规则参与，MVP 只做简单总分或 1-5 分。
- 投票互动：可独立保留，用于人气和曝光。
- 管理员配置评分开放范围：所有访客、报名选手、现场观众或口令用户。
- 系统限制同一来源重复评分。

### 7. 作品展示

活动结束后作品展示页仍然保留，形成长期曝光。

展示重点：
- 封面图。
- 项目名。
- 简介。
- 队伍。
- 赛道。
- 票数 / 奖项。
- Demo / 代码 / 附件链接。

### 8. 飞书同步

本地系统控制业务流程，飞书做备份和协作面板。

MVP 可先做：
- CSV / JSON 导出。
- 同步状态字段。

V1 再做：
- 自动写入飞书多维表格。
- 附件上传飞书云盘。
- 飞书看板状态同步。

## MVP 优先级

P0 必做：
- 管理员登录。
- 活动配置。
- 报名表。
- 个人 / 团队报名。
- 报名审核。
- 作品提交。
- 作品展示。

P1 增强：
- AI 标签和摘要。
- 邮件模板和发送记录。
- 开放评分。
- 投票。
- 数据导出。

P2 扩展：
- 飞书自动同步。
- AI 邮件草稿。
- 组队推荐。
- 归档展示页优化。
- 活动复盘数据面板。

## 技术判断

Java 可以胜任。

实训版最稳组合：
- Servlet 控制请求。
- Service 处理业务流程。
- DAO 通过 JDBC 操作 MySQL。
- JSP 渲染页面。
- Bootstrap 做 UI。

不要一开始就把飞书、AI、邮件都做得很深。先定义接口：

```text
AiService
FeishuSyncService
MailService
FileStorageService
```

早期可以用假实现 / 半自动实现，等核心流程跑通后逐步替换成真实集成。评分模块不再拆评委权限，直接做 `RatingService`。
