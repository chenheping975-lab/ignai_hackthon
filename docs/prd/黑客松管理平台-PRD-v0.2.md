# PRD: 黑客松管理平台 V0.2

## 1. 背景与定位

黑客松管理平台用于支撑一场 AI 黑客松从报名、组队、审核、通知、项目提交、开放评分、投票互动到作品展示的完整流程。

本平台是独立赛事管理系统，和 IGNAI 官网只保持友链关系。平台可复用 IGNAI 的视觉气质和品牌资产，但不承载官网社区首页、文章、成员、记录等内容模块。

## 2. 本次迭代变更

- 移除“评委”独立角色，不做评委账号、评委分配、评委专属评分页。
- 评分改为开放评分：参赛者、现场观众或公开访客都可以按活动规则参与评分。
- 不做复杂评分分级，MVP 只做一个可配置的简单评分维度或总分。
- 视频能力从 MVP 移除：不上传视频、不转码、不播放视频，也不把视频外链作为 MVP 字段。
- 作品展示以封面图、文字简介、Demo 链接、代码链接和附件为主。

## 3. 目标

- 支持 100 人规模线下黑客松的报名、组队、审核和通知。
- 管理员可以配置活动、赛道、报名字段、提交字段和开放评分规则。
- 报名和作品数据本地保存，并可备份 / 同步到飞书。
- AI 辅助整理报名信息、生成标签、摘要和审核建议。
- AI 辅助生成邮件内容，但发送前必须人工确认。
- 支持作品提交、开放评分、投票互动和赛后作品展示。
- MVP 阶段不引入视频相关功能，降低文件和展示复杂度。

## 4. 用户角色

- 访客：浏览赛事信息、作品展示，按规则参与评分或投票。
- 参赛者：报名、组队、提交作品、查看审核和通知，按规则参与评分或投票。
- 队长：管理队伍成员、提交团队作品。
- 管理员：配置活动、审核报名、管理队伍、发送邮件、管理作品、查看数据、管理评分规则。

## 5. MVP 范围

MVP 先支持单场赛事，不做多赛事平台化。

包含：
- 赛事首页
- 个人 / 团队报名
- 管理员配置报名字段
- 报名审核
- AI 标签和摘要
- 邮件模板与人工确认发送
- 队伍管理
- 项目提交
- 作品展示
- 开放评分
- 简单投票
- 本地文件备份
- 飞书导出 / 同步记录

暂不包含：
- 多租户 SaaS
- 完整社区官网
- 评委账号、评委分配、评委专属评分流程
- 复杂评分分级和多轮评审
- 视频上传、视频外链、视频转码、视频在线播放
- 复杂实时协同编辑
- 自动替代管理员做最终审核
- 自动替代用户做最终评分

## 6. 核心流程

### 6.1 赛事配置

管理员创建活动，配置：
- 活动名称、时间、地点、介绍、报名截止时间。
- 赛道列表。
- 奖项列表。
- 报名字段。
- 作品提交字段。
- 开放评分规则。
- 是否开启投票。
- 飞书备份配置。

### 6.2 报名与组队

参赛者选择报名类型：
- 个人报名：填写个人信息、技能、关注赛道、组队需求。
- 团队报名：填写队伍名、队长信息、队员信息、项目方向。

报名提交后：
- 系统保存本地数据库。
- AI 生成标签、摘要和组队建议。
- 管理员审核通过 / 拒绝 / 候补。
- 审核结果通过邮件通知。
- 通过后的数据可同步到飞书。

### 6.3 比赛中提交

队伍提交：
- 队伍名。
- 成员信息。
- 项目标题。
- 赛道。
- 一句话简介。
- 项目说明。
- Demo 链接。
- GitHub / 代码链接。
- PPT / PDF / HTML / ZIP / MP3 / 图片等附件。
- 封面图。

系统保存附件 metadata 和本地文件备份，并记录飞书同步状态。

### 6.4 开放评分与投票

开放评分：
- 所有人按活动规则参与评分，不区分评委和普通用户。
- 管理员可设置评分开放范围：所有访客、报名选手、现场观众或一次性口令用户。
- MVP 建议只做总分或一个简单评分维度，例如 1-5 分。
- 系统记录评分来源标识，避免同一来源重复评分。

投票互动：
- 投票和评分可以合并为一个“支持 / 打分”动作，也可以分别开启。
- 每个作品显示评分均值、评分人数或票数。
- 投票 / 评分限制由管理员配置。
- 活动结束后进入归档模式，关闭评分和投票。

### 6.5 作品展示

作品展示页支持：
- 按赛道筛选。
- 按票数 / 平均分 / 提交时间排序。
- 展示项目封面、名称、简介、团队、赛道、票数 / 平均分。
- 进入详情页查看项目说明、附件、Demo 链接和代码链接。
- 活动结束后切换为归档模式：禁止提交、评分和投票，只展示作品。

## 7. 用户故事

### US-001: 管理员登录后台

**描述：** 作为管理员，我想通过后台账号登录管理系统，以便管理赛事配置和数据。

**验收标准：**
- [ ] 管理员可输入账号密码登录。
- [ ] 登录成功后保存 session。
- [ ] 未登录访问后台时跳转到登录页。
- [ ] 支持退出登录。
- [ ] 密码不能明文存储。

### US-002: 配置赛事基础信息

**描述：** 作为管理员，我想配置赛事名称、时间、地点、赛道和报名截止时间，以便快速生成赛事页面。

**验收标准：**
- [ ] 后台可新增 / 编辑活动信息。
- [ ] 可新增 / 编辑 / 删除赛道。
- [ ] 前台赛事首页展示最新配置。
- [ ] 报名截止后报名按钮变为不可用。

### US-003: 配置报名表单字段

**描述：** 作为管理员，我想配置报名表字段，以便适配不同活动的报名要求。

**验收标准：**
- [ ] 字段支持文本、长文本、单选、多选、日期、文件、链接。
- [ ] 字段可设置必填 / 非必填。
- [ ] 字段可排序。
- [ ] 前台报名页按配置动态展示字段。

### US-004: 个人报名

**描述：** 作为参赛者，我想以个人身份报名，以便参加黑客松并寻找队友。

**验收标准：**
- [ ] 报名者可填写姓名、联系方式、学校 / 公司、技能、关注赛道、组队需求。
- [ ] 提交后生成报名记录。
- [ ] 报名状态默认为待审核。
- [ ] 同一联系方式不能重复报名同一活动。

### US-005: 团队报名

**描述：** 作为队长，我想以团队身份报名，以便一次性提交队伍信息。

**验收标准：**
- [ ] 队长可填写队伍名、队员信息、项目方向和赛道。
- [ ] 系统校验同一成员不能加入多个队伍。
- [ ] 团队报名提交后进入待审核状态。
- [ ] 管理员能查看队伍成员列表。

### US-006: AI 报名标签与摘要

**描述：** 作为管理员，我想让 AI 自动整理报名信息，以便快速判断选手能力和组队需求。

**验收标准：**
- [ ] 报名提交后可触发 AI 标签生成。
- [ ] 标签至少包含技能、赛道、角色倾向、组队需求。
- [ ] AI 生成一段管理员可读摘要。
- [ ] AI 输出可被管理员手动修改。
- [ ] AI 不自动决定审核结果。

### US-007: 报名审核

**描述：** 作为管理员，我想审核报名记录，以便控制参赛名单。

**验收标准：**
- [ ] 管理员可筛选待审核 / 已通过 / 已拒绝 / 候补。
- [ ] 管理员可查看原始表单、AI 标签和摘要。
- [ ] 管理员可批量通过 / 拒绝。
- [ ] 审核动作记录操作人、时间和备注。

### US-008: 邮件通知 Agent

**描述：** 作为管理员，我想让系统根据报名状态生成邮件草稿，以便快速发送审核结果和活动通知。

**验收标准：**
- [ ] 管理员可维护邮件模板。
- [ ] 系统可按通过 / 拒绝 / 候补 / 提醒等场景生成邮件草稿。
- [ ] AI 可根据活动信息和报名信息生成个性化内容。
- [ ] 邮件发送前必须可预览和编辑。
- [ ] 发送后记录发送对象、标题、时间、状态和失败原因。

### US-009: 项目提交

**描述：** 作为队长，我想提交作品资料，以便参加展示、评分和投票。

**验收标准：**
- [ ] 队长可提交项目标题、简介、赛道、说明、Demo 链接、代码链接。
- [ ] 支持上传 PPT / PDF / HTML / ZIP / MP3 / 图片。
- [ ] MVP 不出现视频上传、视频外链或视频播放字段。
- [ ] 文件保存到本地备份目录，数据库保存路径和 metadata。
- [ ] 提交截止后不可修改，除非管理员重新开放。

### US-010: 飞书备份

**描述：** 作为管理员，我想把报名和作品资料备份到飞书，以便团队协作和长期保存。

**验收标准：**
- [ ] 系统记录每条报名 / 作品的飞书同步状态。
- [ ] 同步失败时保留错误信息。
- [ ] 管理员可手动重试同步。
- [ ] MVP 至少支持导出 CSV / JSON 供飞书导入。
- [ ] V1 支持自动同步到飞书多维表格和云盘。

### US-011: 开放评分

**描述：** 作为访客或参赛者，我想给作品打分，以便参与互动并帮助优秀作品获得曝光。

**验收标准：**
- [ ] 管理员可开启 / 关闭开放评分。
- [ ] 管理员可设置评分范围：所有访客、报名选手、现场观众或口令用户。
- [ ] 用户可对作品提交 1-5 分评分。
- [ ] 同一来源不能重复给同一作品评分。
- [ ] 作品页展示平均分和评分人数。
- [ ] 活动归档后评分入口关闭。

### US-012: 作品展示与投票

**描述：** 作为访客，我想浏览作品并投票，以便参与互动和帮助优秀作品获得曝光。

**验收标准：**
- [ ] 作品展示页显示作品卡片。
- [ ] 支持按赛道筛选。
- [ ] 支持投票按钮和票数展示。
- [ ] 活动结束后进入归档模式，投票按钮关闭。
- [ ] 作品详情页展示项目说明、团队信息、附件和外部链接。

## 8. 功能需求

- FR-1: 系统必须支持管理员登录、退出和 session 权限校验。
- FR-2: 系统必须支持单场赛事配置。
- FR-3: 系统必须支持赛道配置。
- FR-4: 系统必须支持报名表单字段配置。
- FR-5: 系统必须支持个人报名。
- FR-6: 系统必须支持团队报名。
- FR-7: 系统必须限制同一参赛者重复加入多个队伍。
- FR-8: 系统必须保存报名原始数据。
- FR-9: 系统必须支持 AI 标签、摘要和审核建议。
- FR-10: 系统必须支持管理员修改 AI 标签。
- FR-11: 系统必须支持报名审核状态流转。
- FR-12: 系统必须支持邮件模板管理。
- FR-13: 系统必须支持 AI 邮件草稿生成。
- FR-14: 系统必须要求邮件发送前人工确认。
- FR-15: 系统必须记录邮件发送日志。
- FR-16: 系统必须支持作品提交。
- FR-17: 系统必须支持文件上传和本地备份。
- FR-18: 系统必须在 MVP 阶段移除视频上传、视频外链和视频播放字段。
- FR-19: 系统必须支持作品展示页。
- FR-20: 系统必须支持作品详情页。
- FR-21: 系统必须支持投票。
- FR-22: 系统必须支持开放评分。
- FR-23: 系统必须支持评分来源限制，避免重复评分。
- FR-24: 系统必须支持飞书导出 / 同步记录。
- FR-25: 系统必须支持基础数据统计：报名人数、通过人数、队伍数、作品数、投票数、评分人数、平均分。

## 9. 非功能需求

- NFR-1: 普通后台列表页 1000 条以内数据查询应在 2 秒内返回。
- NFR-2: 上传文件必须限制类型和大小。
- NFR-3: 后台操作需要权限校验。
- NFR-4: 用户联系方式、邮箱、手机号等个人信息不在公开页展示。
- NFR-5: AI 生成内容必须标记为 AI 建议，不作为最终事实。
- NFR-6: 邮件发送必须记录日志，方便追踪。
- NFR-7: 飞书同步失败不能影响本地报名和提交主流程。
- NFR-8: 公开评分必须做重复提交限制。

## 11. 实施路线

### Phase 1: 本地闭环

- 建库建表。
- 管理员登录。
- 赛事配置。
- 报名和团队报名。
- 后台审核。
- 项目提交。
- 作品展示。

### Phase 2: 互动与自动化

- AI 标签和摘要。
- 邮件模板和发送日志。
- 开放评分。
- 投票。
- CSV / JSON 导出。

### Phase 3: 飞书与传播

- 飞书多维表格同步。
- 飞书云盘附件同步。
- 作品归档页。
- 活动结束复盘数据面板。

## 12. 成功指标

- 100 人报名数据可以完整收集和审核。
- 管理员能在后台筛选、审核、通知参赛者。
- 比赛当天队伍能提交作品资料。
- 所有人可按规则参与开放评分或投票。
- 活动结束后作品展示页可用于公开传播。
- 关键数据本地保存，且能导出或同步到飞书。

## 13. 待确认问题

- 是否需要参赛者账号登录，还是报名后用手机号 / 邮箱 + 验证码查看状态？
- 开放评分是所有访客都能参与，还是仅对报名选手 / 现场观众开放？
- 投票和评分是否合并为一个动作，还是保持两个动作？
- 文件上传大小限制设为多少？建议 MVP 单文件 50MB 以内。
- 飞书优先同步到多维表格，还是先同步到云盘文件夹？
- 邮件服务用学校 / 企业 SMTP，还是第三方邮件服务？
